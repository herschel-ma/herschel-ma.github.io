一次完整的http通信过程

http 通信机制是在一次完整的HTTP通信过程中，web浏览器与web服务器之间将完成下列7个步骤：

  • 建立TCP连接

web浏览器与web服务器首先要通过网络与web服务器建立连接，该连接是通过tcp协议来完成的，该协议与IP协议共同构建Internet，即著名的TCP/IP协议族，因此Internet又称作TCP/IP网络。HTTP是比tcp更高层次的应用层协议，规定只有底层协议建立之后才能建立更高协议的连接，因此，首先要建立TCP连接，TCP连接的端口一般是80。

  • web浏览器向web服务器发送请求信息

一旦建立了tcp连接，浏览器就会向服务器发送请求信息，例如：GET/sample/hello.js/HTTP/1.1

  • web浏览器向web服务器发送请求头信息。

浏览器向服务器发送请求信息之后，一般还要以头信息的方式向浏览器发送一些别的信息，之后浏览器会发送以空白行来告诉服务器发送信息完毕。

  • 服务器返回应答

客户机向服务器发出请求之后，服务器会向客户机发出应答，HTTP/1.1 200 OK, 应答的第一部分是协议的版本号以及应答状态码。

  • web服务器发送应答头信息

正如客户端会随同请求发送关于自身的信息一样，服务器也会随同应答发送自身的数据以及正被请求的文档。

  • web服务器向浏览器发送数据

web服务器向浏览器发送了头信息之后，会发送一个空白行表示头信息的发送结束，接着他就以 Content-Type 应答头信息所描述的数据格式向浏览器发送数据。

  • web服务器关闭TCP连接

一般情况下，一旦web服务器向浏览器发送了请求的数据，就要关闭TCP连接，然后如果浏览器或者服务器的头信息中包含了Connection:keep-alive，TCP连接在发送之后保持连接状态，于是浏览器可以通过相同的连接发送请求，保持连接节省了每个连接建立请求的时间，还节省了网络带宽。
