函数装饰器

作用

  • 函数装饰器实际上也是一个函数，它可以让其它函数在不改变的情况下给其他函数增加额外的功能。装饰器的返回值也是一个函数，它应用的场景是在有切面需求的场景，比如django中的登录认证@login_required, pemission_required,性能测试，事务处理，缓存等等。有了装饰器就可以抽离出大量与函数功能无关的雷同代码并发并继续使用。

import time

def out_time(func):
    def inner_time(*args, **kwargs):
        start_time = time.time()
        func()
        end_time = time.time()
        print("该函数执行的时间是{}秒".format(end_time - start_time))
        return func(*args, **kwargs)
    return inner_time

@out_time
def foo():
    time.sleep(2)
    print('Foo is running.')


if __name__ == "__main__":

    import time
from functools import wraps

def time_it(func):
@wraps(func)  # 实现__name__属性一致
def inner_time(*args, **kwargs):
    start = time.time()
    func()
    end = time.time()
    print("该函数执行的时间是{}".format(end-start))
    return func
return inner_time


@time_it
def foo():
time.sleep(1)
print("foo is running.")
print(foo.__name__)

      # 第一次实现，先梳理一下思路
# 使用yield，每次返回一个数，并且下次返回这个输的位置
# 可以使用循环实现生成器，再遍历输出
def fibbo(num):
a, n, b = 0, 0, 1  # n是迭代次数
while n < num:
yield b  # num>=1，否则无意义
a, b = b, a+b  # 推动迭代
n += 1

for i in fibbo(15):
print(i)  # 直接输出前15个的值

    什么是GIL，有什么用

  • GIL（Global Interpreter Lock）是全局解释器锁，只在Cpython解释器中出现。GIL保证同一时间只有一个线程使用资源，所以python是伪多线程。

    什么是类方法，类实例方法，静态方法

  • 类方法是在类中定义的方法@classmethod，第一个参数是cls, 类的实例和继承类的实例对象可以调用该方法

  • 类实例方法是类中定义的方法，第一个参数是self，该方法只有类实例可以调用
